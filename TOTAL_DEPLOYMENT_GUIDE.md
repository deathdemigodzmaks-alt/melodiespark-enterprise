### OPTION A: VERCEL + RAILWAY (Choose one)

#### Step 1: Push Code
```bash
cd ~/melodiespark  # your project root
git add .
git commit -m "chore: production deployment configs"
git push origin main
```

#### Step 2: Deploy Frontend to Vercel
1. Go to https://vercel.com/new
2. Click "Import Git Repository"
3. Select your melodiespark repo
4. Set project name: `melodiespark`
5. Root directory: `MelodieSpark`
6. Framework: Next.js (auto-detected)
7. Build Command: `pnpm run build` (auto-detected)
8. Environment variables:
   ```
   NEXT_PUBLIC_API_URL=https://melodiespark-api.up.railway.app
   ```
9. Click "Deploy"
10. Wait 2-5 minutes
11. Get URL: `https://melodiespark.vercel.app`

#### Step 3: Deploy API to Railway
1. Go to https://railway.app/new
2. Select "Existing Repository"
3. Select your melodiespark repo
4. Railway auto-detects `apps/api`
5. Add plugins:
   - PostgreSQL 16 (auto-provisions database)
   - Redis 7 (auto-provisions cache)
6. Add environment variables:
   ```
   NODE_ENV=production
   JWT_SECRET=<run: openssl rand -base64 32>
   ANTHROPIC_API_KEY=sk-...
   OPENAI_API_KEY=sk-...
   DATABASE_URL=<auto-generated>
   REDIS_URL=<auto-generated>
   ```
7. Click "Deploy"
8. Wait 3-5 minutes
9. Get URL: `https://melodiespark-api.up.railway.app`

#### Step 4: Update Cross-Environment Variables
1. In Vercel settings, update:
   ```
   NEXT_PUBLIC_API_URL=https://melodiespark-api.up.railway.app
   ```
2. Re-deploy Vercel

#### Step 5: Verify
```bash
curl https://melodiespark-api.up.railway.app/health
# Should return: {"status":"ok","timestamp":"...","version":"1.0.0"}

open https://melodiespark.vercel.app
# Should load frontend
```

**Time:** 15 minutes | **Cost:** $30-50/month | **Result:** Live & automated ✅

### OPTION B: SELF-HOSTED VPS (Choose one)

#### Step 1: Provision VPS
Create new Ubuntu 22.04 instance on:
- DigitalOcean ($6/month)
- Linode ($5/month)
- AWS EC2 ($10-20/month)
- Vultr ($2.50/month)

Note: IP address, username (usually `root`)

#### Step 2: Run Deployment Script
```bash
cd MelodieSpark
bash deploy-vps.sh
```

Script will prompt for:
- VPS IP address
- VPS username
- Domain name (e.g., melodiespark.com)
- Email for SSL notifications

Script will automatically:
- Install Docker & Compose
- Clone your repository
- Build services
- Start services
- Configure Nginx
- Setup SSL with Let's Encrypt
- Enable auto-renewal
- Setup backups
- Setup monitoring

#### Step 3: Point Domain to VPS
1. Get VPS IP from provider
2. Go to your domain registrar
3. Add DNS A record:
   ```
   melodiespark.com    A    <VPS_IP>
   www.melodiespark.com A    <VPS_IP>
   ```
4. Wait 15-60 minutes for DNS propagation

#### Step 4: Verify
```bash
# Wait for SSL certificate (usually 5-10 minutes)
curl https://melodiespark.com/
# Should show your frontend

curl https://melodiespark.com/api/health
# Should return health check
```

**Time:** 30 minutes | **Cost:** $6-20/month | **Result:** Full control & ownership ✅

### OPTION C: KUBERNETES (Choose one)

#### Step 1: Create Kubernetes Cluster
```bash
# GKE (Google)
gcloud container clusters create melodiespark --zone us-central1-a

# EKS (AWS)
eksctl create cluster --name melodiespark --region us-east-1

# DigitalOcean
doctl kubernetes cluster create melodiespark --region nyc3
```

#### Step 2: Install Required Components
```bash
# NGINX Ingress Controller
helm repo add ingress-nginx https://kubernetes.github.io/ingress-nginx
helm install ingress-nginx ingress-nginx/ingress-nginx

# cert-manager (for SSL)
helm repo add jetstack https://charts.jetstack.io
helm install cert-manager jetstack/cert-manager \
  --set installCRDs=true
```

#### Step 3: Create Namespace & Secrets
```bash
kubectl create namespace production

kubectl create secret generic melodiespark-secrets \
  --from-literal=database-url="postgresql://user:pass@postgres:5432/melodiespark" \
  --from-literal=redis-url="redis://redis:6379" \
  --from-literal=jwt-secret="$(openssl rand -base64 32)" \
  --from-literal=anthropic-api-key="sk-..." \
  --from-literal=openai-api-key="sk-..." \
  -n production
```

#### Step 4: Deploy Application
```bash
# Update image URLs in k8s/deployment.yaml
# Replace YOUR_ORG with your Docker registry org

kubectl apply -f k8s/deployment.yaml

# Verify deployment
kubectl get pods -n production
kubectl get svc -n production
kubectl get ingress -n production
```

#### Step 5: Setup DNS
Point your domain to the Ingress IP:
```bash
kubectl get ingress -n production
# Note the EXTERNAL-IP, add as DNS A record
```

#### Step 6: Verify
```bash
curl https://melodiespark.com/health
curl https://api.melodiespark.com/health
```

**Time:** 1-2 hours | **Cost:** $50+/month | **Result:** Enterprise scale ✅

## 🆘 Troubleshooting

### Services Won't Start Locally
```bash
# Check Docker
docker ps
docker-compose ps

# View logs
docker-compose logs

# Rebuild from scratch
docker-compose down -v
docker-compose build --no-cache
docker-compose up -d
```

### Vercel/Railway Deployment Fails
1. Check build logs in dashboard
2. Ensure `.env` variables are set
3. Verify package scripts in package.json
4. Try manual build: `pnpm build`

### VPS Deployment Fails
```bash
# SSH into VPS
ssh root@<VPS_IP>

# Check Docker
docker ps
docker-compose ps -a

# View logs
docker-compose logs -f api

# Restart services
docker-compose restart
```

### Kubernetes Deployment Issues
```bash
# Check pod status
kubectl get pods -n production

# View pod logs
kubectl logs -n production <pod-name>

# Describe pod for errors
kubectl describe pod -n production <pod-name>

# Check events
kubectl get events -n production
```
